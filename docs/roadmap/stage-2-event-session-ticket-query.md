# 阶段 2：活动、场次、票种建模与公开查询接口

返回主路线图：[活动预约与票务平台建议路线图](./event-booking-roadmap.md)

## 目标

完成 Event、Session、TicketType 的核心建模，并提供面向用户侧的活动列表与详情查询能力。

## 范围

- 活动状态。
- 场次时间窗口。
- 票种价格与库存字段设计。
- 基础列表/详情接口。
- 请求/响应 DTO 设计。

## 非范围

- 后台管理写接口。
- 下单与支付。

## 要求

- 明确表结构。
- 明确状态枚举。
- 明确接口字段。
- 明确上架/下架语义。
- 明确边界条件。
